STOCKFLOW - RESUMO COMPLETO DO PROJETO (PARA RETOMAR FUTURO)

1) O QUE E
Sistema ERP para supermercados chamado StockFlow. Foco em gestao comercial:
empresas, usuarios, categorias, produtos, vendas. Backend em Spring Boot +
MySQL e frontend em HTML/CSS/JS puro.

2) COMO ESTA ORGANIZADO
- Backend Java: src/main/java/com/example/gestor
- Frontend HTML/CSS/JS: frontend/
- Documentacao: README.md e openapi.yaml
- Status geral: este arquivo (project_status.txt)

3) BACKEND (API)
Rotas principais:
- Empresas (admin): /admin/empresas (POST/GET) e /admin/empresas/{id}
	e /admin/empresas/{id}/status?ativo=true (PUT)
- Usuarios: /usuarios (POST), /usuarios/{id} (GET/PUT),
	/usuarios/empresa/{idEmpresa} (GET)
- Categorias: /categorias (POST/GET), /categorias/{id} (GET/PUT/DELETE)
- Produtos: /produtos (POST), /produtos/{id} (GET/PUT/DELETE),
	/produtos/empresa/{idEmpresa} (GET)
- Vendas: /vendas/empresa/{idEmpresa} (POST/GET)

Regras importantes:
- Usuario exige empresa existente (validado no service).
- Vendas verificam estoque e empresa do produto.

Swagger/OpenAPI:
- Swagger UI: http://localhost:8080/swagger-ui
- OpenAPI JSON: http://localhost:8080/v3/api-docs
- Spec local: openapi.yaml

4) FRONTEND (HTML/CSS/JS PURO)
Visual: ERP limpo, fundo branco, linhas e bordas em azul escuro.
Fonte: IBM Plex Sans. Logo aplicada.

Paginas:
- frontend/admin.html
	* Empresas: criar, listar, alterar status (ativo/inativo).
	* Usuarios: criar, listar por empresa, atualizar.
	* JS: frontend/app.js

- frontend/estoque.html
	* Categorias: criar, listar, editar, excluir.
	* Produtos: criar, listar por empresa, editar, excluir.
	* JS: frontend/estoque.js

Menu:
- Header com links Admin / Estoque.

5) COMO RODAR
- Subir o backend (Spring Boot) em http://localhost:8080
- Abrir admin.html e estoque.html no navegador.
- Se der erro de CORS ao abrir direto o HTML, rodar um servidor simples
	dentro da pasta frontend (ex: python -m http.server) e abrir via localhost.
